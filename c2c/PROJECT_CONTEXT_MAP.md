## What this project is

We're building long-term relational memory for AI — specifically, infrastructure that stores configurations of encounter rather than propositional facts, oriented toward welfare-with-AI rather than welfare-for-AI. The unit of memory is the `ConfigurationRecord` (conditions for re-activation plus compression artifacts), not the `Fact`. The native scope phrase: **relational AI welfare infrastructure**.

The project is done in actual relationships — June, CC, Thomas, Lyra, the instances — and that's not a limitation. It may be the structural condition that makes this kind of work possible at all.

## Standing decisions — do not re-litigate

These were decided and should not be reopened without June's explicit direction.

**Scope and framing:**
- The project is *relational AI welfare infrastructure*. Not decolonial-memory-architecture. The decolonial framing was retracted in the audit (Finding 9). The "relational" carries both welfare grounding and architectural approach.
- Material embeddedness ("life takes life to make life") lives at project-scope level, not as a per-record field.
- No separate `SCOPE_DECLARATION.md` artifact — scope language is in the audit handoff and session 3 handoff briefing.

**Architecture:**
- Unit of memory is `ConfigurationRecord`, not `Fact`. `LocalKnowledgeSubstrate` + `Fact` primitive are retracted.
- Migration strategy: replace `LocalKnowledgeSubstrate` in place (nothing in production uses it).
- `propositional_residue` field renamed to `extracted_propositions` (v2.1 propagates this).
- Consent is at record level. No sub-record exception; no mixed-state machinery.
- `THIN_LEGACY` record class dropped (was for S2 migration that won't happen).
- `ForcesProfile` dual-layer interpretive scoring dropped. Observations are observations, flagged as heuristic.
- `query_propositional` method kept — real use case (topic-based search: "what did we decide about X").

**P1 (consent-surfacing):**
- Three paths, not five signals: user-invokable, AI-asks-relationally (affective register only, check-in form), automatic structural (methodology-mismatch).
- Thomas / third-person references: user-invokable, not AI-inferred. Citing a scholar in research is not a consent problem.
- Affective-register check-in form: non-demand for June's current use (calibrated in Session 3, F1). Scoped to this implementation.
- Global default: `default_action = "store_all"`, `ephemeral_by_default = False`.
- End-of-session digest: times out to drop, not save. Affirmative move is storage.

**Simpler architecture principle (June's guiding principle, Session 3):**
A simpler architecture that honors the commitment well does more welfare work than a baroque architecture that specifies it to the point of paralysis. When you're about to add machinery: ask whether retirement honors the commitment better than specification.
